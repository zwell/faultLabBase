# 任务：FaultLab 底座业务状态面板

## 背景

FaultLab 是一个故障排查训练平台。用户启动电商底座后，
会向底座注入一个故障（如 MySQL 连接池打满），然后通过终端排查根因。

业务状态面板的作用是：**注入故障前后，让用户直观看到哪条链路出现异常**，
知道「出事了」之后再去终端排查具体原因。

---

## 要实现的功能

### 1. 概要面板（默认展示）

三张卡片，横向排列，每张对应一条业务链路：

**下单链路**
- 请求量（次/分钟）
- P99 延迟（ms）
- 错误率（%）

**消息链路**
- Consumer Lag（条）
- 处理耗时（ms）

**存储层**
- MySQL 连接数（当前 / 上限）
- Redis 内存使用率（%）

每张卡片有状态色：
- 正常：默认暗色（与页面背景融合）
- 警告：黄色边框 + 黄色数值
- 异常：红色边框 + 红色数值

阈值（可配置，先硬编码）：
```js
const THRESHOLDS = {
  error_rate:       { warn: 0.05, critical: 0.15 },   // 5% / 15%
  p99_ms:           { warn: 500,  critical: 2000 },    // 500ms / 2s
  consumer_lag:     { warn: 100,  critical: 500 },
  mysql_conn_ratio: { warn: 0.7,  critical: 0.9 },     // 连接数/上限
  redis_mem_ratio:  { warn: 0.7,  critical: 0.9 },
}
```

### 2. 详情抽屉（点击卡片展开）

点击任意卡片，在卡片下方展开详情区域（不跳新页面）：

**下单链路详情**
- 请求量趋势（最近 2 分钟，折线图，每 3 秒一个点）
- P99 / P95 / P50 延迟
- 2xx / 4xx / 5xx 各自数量

**消息链路详情**
- Lag 趋势（最近 2 分钟折线图）
- 最近 10 条消费记录（offset、耗时）

**存储层详情**
- MySQL：连接数趋势、当前活跃查询数（SHOW PROCESSLIST 结果条数）
- Redis：内存使用量（MB）、命中率（%）、当前 key 数量

### 3. 数据刷新

- 每 3 秒自动刷新一次
- 刷新时数值平滑过渡（不闪烁）
- 右上角显示「最后更新时间」
- 底座未运行时面板显示灰色占位，不报错

## 后端 API 约定

前端调用以下接口（后端已实现或需同步实现）：

```
GET /api/basecamps/:id/metrics/summary
返回：
{
  "order": {
    "requests_per_min": 42,
    "p99_ms": 120,
    "error_rate": 0.01
  },
  "consumer": {
    "lag": 8,
    "avg_process_ms": 102
  },
  "storage": {
    "mysql_connections": 12,
    "mysql_connections_max": 151,
    "redis_mem_used_mb": 24,
    "redis_mem_max_mb": 512,
    "redis_hit_rate": 0.94
  },
  "collected_at": "2024-01-01T00:00:00.000Z"
}

GET /api/basecamps/:id/metrics/detail?chain=order&limit=40
返回最近 40 个数据点（用于折线图）：
{
  "points": [
    { "ts": "2024-01-01T00:00:00.000Z", "p99_ms": 120, "requests": 7, "errors": 0 },
    ...
  ]
}
```

如果后端接口还没实现，先用 mock 数据开发，mock 数据要能模拟「正常 → 故障」的变化过程。

## 视觉要求

- 延续现有页面风格：深色背景（#0d1117）、卡片略亮背景
- 不引入新的 UI 组件库，用现有项目已有的组件
- 折线图用 Chart.js 或现有项目已引入的图表库，不新增依赖
- 数值字体稍大（1.5rem），单位字体小（0.75rem），形成层次
- 卡片高度固定（约 120px），详情区展开时卡片本身不变形

---

## 交互细节

- 同时只能展开一个详情区（点击已展开的卡片则收起）
- 详情区展开/收起有过渡动画（200ms）
- 数值从上一次到这一次有变化时，短暂高亮（flash 效果，500ms）
- 底座未运行时，卡片显示「— —」占位，不显示 0

---

## 组件位置

在底座详情页的「启动/停止控制区」和「容器终端」之间插入此面板。
面板标题：「业务状态」，右侧显示刷新状态。

---

## 不需要做的

- 不需要历史数据持久化（刷新页面数据重置）
- 不需要告警通知（邮件/webhook 等）
- 不需要自定义阈值的 UI
- 不需要导出数据功能
